# Руководство пользователя FOCAL

Этот проект поддерживает учебное подмножество языка FOCAL. Основной целевой
интерпретатор работает в RARS, а Python-версия используется как эталон и удобная
среда для проверки программ.

## Формат программы

Каждая строка программы имеет номер:

```focal
1: SET A=10
2: TYPE "A = ",A,!
3: QUIT
```

Строки выполняются по возрастанию номеров, если выполнение не изменено через
`GOTO`, `IF ... THEN` или `IF ... GOTO`.

## Запуск Python REPL

```bash
python -m focal_interpreter.main
```

Пример сессии:

```text
> 10 SET A=2+3*4
> 20 TYPE "A = ",A,!
> 30 QUIT
> LIST
10: SET A=2+3*4
20: TYPE "A = ",A,!
30: QUIT
> RUN
A = 14.0
```

Команды REPL:

| Команда | Действие |
|---|---|
| `RUN` или `GO` | запустить программу с первой строки |
| `RUN N` или `GO N` | запустить программу со строки `N` |
| `LIST` | показать введенную программу |
| `ERASE` | очистить программу |
| `HELP` | показать краткую справку |
| `QUIT` | выйти из REPL |

Строка с номером добавляет или заменяет строку программы. Пустая строка с
номером удаляет строку:

```text
> 20
```

Строка без номера выполняется сразу как однострочная команда.

## Команды языка

### SET

Присваивание переменной или массиву:

```focal
10 SET A=5
20 SET B=A*2
30 SET X(0)=42
```

### TYPE

Вывод строк и чисел. `!` означает перевод строки.

```focal
10 TYPE "Hello",!
20 TYPE "A = ",A,!
30 TYPE "Sum = ",A+B,!
```

### ASK

Ввод числа:

```focal
10 ASK N, "N = "
20 TYPE "You typed ",N,!
```

### IF

Условное выполнение:

```focal
10 IF A>B DO TYPE "A is greater",!
20 IF A=0 THEN 100
30 IF A<0 GOTO 200
```

### GOTO

Переход к строке:

```focal
10 GOTO 100
```

### FOR

Цикл со счетчиком:

```focal
10 FOR I=1,5 DO TYPE I,!
20 FOR I=1,5 DO SET S=S+I
```

В Python-версии поддерживается шаг:

```focal
10 FOR I=10,1,-1 DO TYPE I,!
```

В RARS-версии сейчас поддержан базовый вариант с шагом `+1`.

### DO

В Python-версии выполняет диапазон строк:

```focal
10 DO 100,120
```

## Выражения

Поддерживаются:

- числа;
- переменные `A-Z`;
- массивы `A(i)`;
- скобки;
- `+`, `-`, `*`, `/`, `^` в Python-версии;
- унарный минус;
- сравнения `=`, `<>`, `<`, `>`, `<=`, `>=`;
- логические `AND`, `OR`, `NOT` в Python-версии.

Примеры:

```focal
10 SET A=2+3*4
20 SET B=(2+3)*4
30 IF A<B THEN 100
```

## Встроенные функции

Python-версия поддерживает:

| Функция | Значение |
|---|---|
| `FSQT(x)` | квадратный корень |
| `FABS(x)` | модуль |
| `FEXP(x)` | экспонента |
| `FLOG(x)` | натуральный логарифм |
| `FSIN(x)` | синус |
| `FCOS(x)` | косинус |
| `FATN(x)` | арктангенс |
| `FITR(x)` | целая часть |
| `FRND(x)` | округление |
| `FRAN()` | случайное число |

В RARS-версии функции пока не входят в минимальный набор.

## RARS REPL

Если открыть `rars_focal_interpreter.asm` в RARS и запустить его, интерпретатор
стартует в интерактивном режиме:

```text
FOCAL/RARS REPL. Enter numbered lines, LOAD, SAVE, RUN, ERASE, QUIT.
> 10 SET A=2+3*4
> 20 TYPE "A = ",A,!
> 30 QUIT
> RUN
A = 14.0
```

Команды RARS REPL:

| Команда | Действие |
|---|---|
| FOCAL-команда без номера | выполнить сразу, не сохраняя в программе |
| `RUN` или `GO` | скомпилировать введенные строки в байткод и выполнить |
| `LIST` | вывести текущий буфер программы |
| `LOAD <file>` | загрузить FOCAL-программу из файла в буфер |
| `SAVE <file>` | сохранить текущий буфер программы в файл |
| `ERASE` | очистить буфер программы |
| `QUIT` | выйти из интерпретатора |

Строки с номерами хранятся как программа:

- повторный ввод того же номера заменяет строку;
- строка, содержащая только номер, удаляет строку;
- `LIST` и `RUN` используют порядок возрастания номеров, даже если строки
  вводились в другом порядке.

Пример редактирования:

```text
> 20 TYPE "old",!
> 20 TYPE "new",!
> 20
```

Пример немедленного выполнения:

```text
> TYPE 2+3*4,!
14.0
> SET A=5
> TYPE A*2,!
10.0
```

Пример работы с файлом:

```text
> LOAD C:\Users\Admin\Desktop\Focal simulator\demo\rars\hello.focal
> LIST
> RUN
> SAVE C:\Users\Admin\Desktop\Focal simulator\saved_hello.focal
```

Путь к файлу передается после команды без кавычек. В RARS GUI рекомендуется
указывать полный путь к `.focal`-файлу, потому что относительный путь зависит от
папки запуска Java/RARS.

## Текущие ограничения RARS-версии

- исходные числовые литералы пока целые;
- вычисления и ввод/вывод выполняются как float;
- `FOR` поддержан только с шагом `+1`;
- встроенные математические функции пока не реализованы;
- `LOAD` читает файл целиком в буфер до 8191 символа;
- максимальная длина строки в RARS REPL — 127 символов;
- максимум строк в RARS REPL — 128.
