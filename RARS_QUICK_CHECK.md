# Быстрая проверка интерпретатора FOCAL в RARS

Этот файл описывает, как вручную проверить, что FOCAL-интерпретатор запускается
и работает **внутри симулятора RARS**.

## 1. Запуск

1. Открыть RARS.
2. Открыть файл:

```text
rars_focal_interpreter.asm
```

3. Нажать `Assemble`.
4. Нажать `Run`.
5. В нижней панели открыть вкладку `Run I/O`.

После запуска должен появиться интерактивный режим:

```text
FOCAL/RARS REPL. Enter HELP for commands.
>
```

## 2. Минимальная программа

Ввести в `Run I/O`:

```text
10 SET A=2+3*4
20 TYPE "A = ",A,!
30 QUIT
LIST
RUN
```

После `LIST` ожидается:

```text
10 SET A=2+3*4
20 TYPE "A = ",A,!
30 QUIT
```

После `RUN` ожидается:

```text
A = 14.0
```

## 3. Команды REPL

| Команда | Что делает |
|---|---|
| строка с номером | добавляет или заменяет строку программы |
| FOCAL-команда без номера | выполняется сразу, без сохранения в программе |
| только номер строки | удаляет строку программы |
| `LIST` | показывает текущую программу |
| `LOAD <file>` | загружает FOCAL-программу из файла в буфер REPL |
| `SAVE <file>` | сохраняет текущий буфер REPL в файл |
| `RUN` или `GO` | компилирует введенную программу в байткод и запускает VM |
| `ERASE` | очищает программу |
| `HELP` | показывает справку по командам REPL |
| `QUIT` | завершает интерпретатор |

Команды REPL можно вводить в любом регистре: `help`, `load`, `save`, `run`.

Строки можно вводить не по порядку: `LIST` и `RUN` используют порядок
возрастания номеров.

Быстрая проверка, что REPL именно считает интерактивно:

```text
TYPE 2+3*4,!
SET A=5
TYPE A*2,!
```

Ожидается:

```text
14.0
10.0
```

## 4. Дополнительные проверки

### Замена строки

```text
ERASE
10 SET A=1
20 TYPE "old",!
20 TYPE "new",!
30 QUIT
LIST
RUN
```

Ожидается:

```text
10 SET A=1
20 TYPE "new",!
30 QUIT
new
```

### Условие IF

```text
ERASE
10 SET A=5
20 IF A=5 THEN 40
30 TYPE "bad",!
40 TYPE "ok",!
50 QUIT
RUN
```

Ожидается:

```text
ok
```

### Массивы

```text
ERASE
10 SET A(0)=10
20 SET A(1)=20
30 TYPE "Sum = ",A(0)+A(1),!
40 QUIT
RUN
```

Ожидается:

```text
Sum = 30.0
```

### Цикл FOR

```text
ERASE
10 SET S=0
20 FOR I=1,5 DO SET S=S+I
30 TYPE "S = ",S,!
40 QUIT
RUN
```

Ожидается:

```text
S = 15.0
```

### Ввод ASK

```text
ERASE
10 ASK A, "A = "
20 TYPE "A*2 = ",A*2,!
30 QUIT
RUN
```

Когда интерпретатор напечатает:

```text
A =
```

ввести:

```text
7
```

Ожидается:

```text
A*2 = 14.0
```

### Загрузка и сохранение файла

В RARS GUI лучше использовать полный путь к файлу. Относительный путь зависит от
того, из какой папки был запущен RARS, поэтому команда может вернуть `File error`.

Сначала сохранить текущую программу:

```text
ERASE
10 TYPE "from file",!
20 QUIT
SAVE C:\Users\Admin\Desktop\Focal simulator\saved.focal
ERASE
LOAD C:\Users\Admin\Desktop\Focal simulator\saved.focal
LIST
RUN
```

Ожидается:

```text
Saved
Loaded
10 TYPE "from file",!
20 QUIT
from file
```

Также можно загрузить готовые демо из папки проекта, например:

```text
LOAD C:\Users\Admin\Desktop\Focal simulator\demo\rars\hello.focal
LIST
RUN
```

## 5. Что реализовано

В RARS-интерпретаторе реализованы:

- интерактивный REPL;
- хранение программы по номерам строк;
- немедленное выполнение FOCAL-команд без номера строки;
- `LIST`, `LOAD`, `SAVE`, `RUN`/`GO`, `ERASE`, `QUIT`;
- компиляция FOCAL-программы во внутренний байткод;
- выполнение байткода стековой VM;
- переменные `A-Z`;
- массивы `A(i)`;
- команды `SET`, `TYPE`, `ASK`, `IF ... THEN`, `IF ... DO`, `GOTO`, `FOR`, `QUIT`;
- выражения со скобками, унарным минусом, `+`, `-`, `*`, `/`;
- сравнения `=`, `<>`, `<`, `>`, `<=`, `>=`.

## 6. Ограничения текущей версии

- Реализовано учебное подмножество FOCAL, а не полный исторический FOCAL.
- Числовые литералы в RARS REPL пока целые: `10`, `-5`, `2+3*4`.
- Внутри VM значения хранятся и печатаются как float, поэтому вывод вида
  `14.0`.
- `FOR` поддерживает только шаг `+1`.
- Встроенные математические функции FOCAL (`FSQT`, `FSIN`, `FCOS` и т.п.) в
  RARS-версии пока не реализованы.
- `LOAD` читает файл целиком в буфер до 8191 символа; файл должен содержать
  обычные нумерованные строки FOCAL.
- Максимум строк в RARS REPL: 128.
- Максимальная длина одной строки в RARS REPL: 127 символов.

## 7. Смысл текущей реализации

Это уже не трансляция `.focal` в отдельный `.asm`. Сценарий работы такой:

```text
RARS запускает rars_focal_interpreter.asm
-> пользователь вводит FOCAL-строки в REPL
-> RUN
-> интерпретатор внутри RARS компилирует FOCAL в байткод
-> VM внутри RARS выполняет байткод
```
